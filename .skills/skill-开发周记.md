# Skill: 开发周记

## 路由判定

### 应该读这个 Skill 的场景
- 每周结束时需要总结本周工作
- 需要向上级或团队汇报开发进度
- 需要回顾本周完成、未完成、待办事项
- 需要记录关键决策和技术债

### 不应该读这个 Skill 的场景
- 需要查看具体模块的 API 或数据模型（应读对应模块 Skill）
- 日常开发中不需要总结汇报

## 目标

生成一份结构化的开发周记，汇总本周的工作内容。周记的作用：
- 让阅读者快速了解本周进展（不要求事先了解项目）
- 记录关键决策及其原因、影响
- 标记未完成事项和技术债

## 输出格式

Markdown 表格 + 分级标题，按日期倒序或维度组织。

### 必须包含的内容

| 区块 | 内容要求 |
|------|---------|
| 本周总览 | 阶段、主要成果、涉及模块 |
| 每日记录 | 日期 + 当天完成的核心变更，每个变更附文件路径和简要说明 |
| Bug 修复 | 问题表现 → 根因定位 → 修复方案（含文件、行号） |
| 关键决策 | 决策内容 → 原因 → 影响 |
| 未完成事项 | 待办列表（checkbox），含已知问题、遗留 TODO、下一步计划 |

### 信息来源

综合以下来源生成周记：
1. `git log` — 本周提交记录
2. 工作区未提交变更（`git status` + `git diff`）
3. `AGENTS.md` — 项目架构和模块索引
4. `.skills/` 目录下的变更记录
5. `details/` 目录下的 API 和数据模型变更

### 写作规范

- 每条变更附文件路径
- Bug 修复写清楚根因（哪几行代码导致的），而非只写"修复了 XX"
- 决策记录写清楚 trade-off 而非只写结论
- 未完成事项写清楚阻塞原因
- 语言：中文，技术术语可保留英文
- 客观陈述，不需要"本周努力工作了"等主观评价
